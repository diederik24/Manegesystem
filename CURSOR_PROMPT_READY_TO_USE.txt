Ik moet de Manegeplan API integreren in deze webapp zodat klanten hun eigen data kunnen bekijken via hun persoonlijke API key.

API DETAILS:
- Endpoint: GET https://jouw-domein.vercel.app/api/get-customer-data
- Authenticatie: Header 'X-API-Key' met de API key van de klant
- Elke klant heeft een eigen unieke API key

WAT MOET ER GEBEUREN:

1. API KEY LOGIN
   - Component waar klant hun API key kan invoeren
   - Opslaan in localStorage (of andere state management)
   - Validatie van API key format

2. DATA OPHALEN
   - Service functie die de API aanroept met de API key
   - Error handling voor:
     * 401 - Ongeldige/missing API key
     * 404 - Klant niet gevonden  
     * Netwerkfouten
   - Loading states tijdens API calls

3. DASHBOARD COMPONENT
   Toon de volgende data duidelijk:
   
   BELANGRIJKSTE INFO:
   - Hoeveel lessen nog op leskaart staan (totaalResterendeLessen)
   - Welke lessen ze rijden (lessons array):
     * Les naam
     * Dag van de week
     * Tijd
     * Instructeur naam
   
   ANDERE INFO:
   - Persoonlijke gegevens (naam, email, telefoon)
   - Huidig saldo (balance)
   - Leskaarten details (per kaart: resterendeLessen, eindDatum, status)
   - Openstaande facturen/transacties
   - Gezinsleden (als van toepassing)

IMPLEMENTATIE STRUCTUUR:

1. api/manegeplan.ts (of .js)
   - Functie: getCustomerData(apiKey: string)
   - Error handling
   - Type definitions voor response

2. hooks/useCustomerData.ts (of .js)
   - React hook die data ophaalt
   - Loading state
   - Error state
   - Auto-refresh optie (optioneel)

3. components/ApiKeyLogin.tsx
   - Input veld voor API key
   - Validatie
   - Opslaan in localStorage
   - Error messages

4. components/CustomerDashboard.tsx
   - Toon alle klantdata
   - Focus op resterendeLessen en lessons
   - Cards/sections voor verschillende data types
   - Loading spinner
   - Error states

REQUIREMENTS:
- TypeScript voor type safety
- Moderne React patterns (hooks, functional components)
- Responsive design
- Loading states
- Error handling
- Gebruiksvriendelijke UI
- Duidelijke weergave van resterendeLessen en lessen

EXTRA (OPTIONEEL):
